# Your first spec

A Catenator spec does not require a complete system description to be useful. It requires enough description that an agent can begin — and enough structure that the description can grow without breaking.

This section walks through writing a spec from scratch. It starts with five fields. It adds views. It connects vocabulary. It validates. By the end, you have a spec that is minimal, valid, and buildable.

---

## Before you start

You need one thing: a sentence that describes what you are building.

Not a vision statement. Not a feature list. A sentence that names the system and its purpose. SIC is a satirical publishing imprint that produces card-format definitions of business jargon. That is enough to begin.

If you cannot write that sentence, the spec is premature. Write the sentence first.
