# Referencing vocabulary

A spec gains precision when it draws from a declared vocabulary. A term defined in a vocabulary carries its full definition — descriptor, type, attributes, relationships, constraints — into every spec that references it. The spec does not need to redefine it. It references it by name.

---

## Declaring a vocabulary

Vocabulary is declared in the provenance block:

```yaml
spec:
  subject: SIC publishing platform
  author: Parth Upadhye
  organization: Engaged Inquiry
  version: 1
  date: 2026-03-09
  vocabulary:
    - catenator-vocab/specimens/sic
```

The vocabulary reference names the specimen file the spec draws from. Any term defined in that specimen can be used in the spec without redefinition. Any agent reading the spec can load the vocabulary and resolve every term.

---

## Using vocabulary terms in a spec

Once a vocabulary is declared, terms from it are referenced by name:

```yaml
model:
  - $ref: manuscript
  - $ref: proof
  - $ref: edition
```

The `$ref` key tells the agent that this term is defined in the declared vocabulary. The agent resolves the reference and has access to the full term definition — attributes, relationships, constraints — without the spec needing to repeat them.

A term used without a `$ref` is an inline definition — valid, but local to this spec. An inline term MUST include at minimum a `descriptor`, a `type`, and a `definition`:

```yaml
model:
  - name: card_pack
    descriptor: model
    type: product
    definition: >
      A packaged set of SIC cards sold as a single unit.
    attributes:
      - card_count: integer
      - theme: string
```

---

## Mixing referenced and inline terms

A spec can use both referenced and inline terms. Referenced terms are the vocabulary's definitions. Inline terms are local extensions — things the system requires that the vocabulary does not yet define.

When an inline term would be useful across multiple specs, it is a candidate for contribution to the vocabulary. The vocabulary grows from the specs that use it.

---

## When no vocabulary is declared

A spec with no vocabulary declaration is valid. Every term it uses must be defined inline. This is appropriate for:

- Early-stage specs where the domain vocabulary is not yet established
- Experimental specs that are exploring a new domain
- Minimal specs where the system is simple enough that inline definitions are not a burden

A spec without a vocabulary declaration MUST NOT use `$ref` for any term. Every term must be self-contained.

**A vocabulary makes a spec portable. A referenced term means the same thing everywhere.**
