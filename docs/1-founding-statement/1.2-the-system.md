# The system

The subject of every Catenator spec is a system.

A system is whatever is being designed and built. It can be large or small, an entire platform or a single workflow, a physical operation or a digital product. The spec describes it at whatever level of resolution the work requires.

The word "system" is deliberate. A product is an output — something delivered to someone. A system is the thing being constructed — the architecture, the logic, the rules, the parts and how they connect.

You build a system. You deliver a product. A Catenator spec is about the former.

A system has no minimum size. A spec can describe:

- an entire pharmacy platform
- a single dispensing workflow within it
- a drug interaction check within that workflow

Each is a system at a different level of resolution. Each can have its own spec. Specs at lower levels of resolution can be referenced by specs at higher levels — this is composability.

A system also has no required domain. The same spec format describes a SaaS platform, a grain elevator, a broadcast network, or a publishing imprint. The domain vocabulary changes. The framework does not.

**The spec is about the system. Everything else specifies it.**
