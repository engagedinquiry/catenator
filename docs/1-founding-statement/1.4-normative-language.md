# Normative language

Catenator uses the normative vocabulary established by RFC 2119. Three keywords carry precise meaning throughout the standard:

`MUST` — an absolute requirement. A conforming spec has no discretion. Deviation means the spec is not a valid Catenator spec.

`SHOULD` — a strong recommendation. A conforming spec ought to follow this unless there is a documented reason not to. Deviation is permitted but must be deliberate.

`MAY` — an option. A conforming spec can include this or not. No justification required either way.

These keywords appear in uppercase throughout the standard wherever a requirement, recommendation, or option is stated. When they do not appear, the statement is explanatory — context, not obligation.

`CAN` is not a normative keyword. When it appears in the standard it describes capability, not permission.
