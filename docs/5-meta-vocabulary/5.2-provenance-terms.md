# Provenance terms

The language of identifying and tracking specs.

| Term           | Defines                                                      |
| -------------- | ------------------------------------------------------------ |
| `author`       | the person who wrote the spec                                |
| `organization` | the entity under whose authority the spec is written         |
| `publisher`    | the entity that makes the spec available                     |
| `publication`  | the act or artifact of making the spec available             |
| `version`      | which iteration of the spec this is — versions are immutable |
| `date`         | when this version was written or published                   |

Every Catenator spec MUST include `author`, `version`, and `date`. A spec without these is not a valid Catenator spec.

---

**The meta-vocabulary is the stud. It is what makes composability possible across specs, domains, tools, and agents.**
