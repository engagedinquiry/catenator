# Meta-vocabulary

The meta-vocabulary is the language of specs. It is distinct from the language of systems.

The descriptors name what is being specified. The meta-vocabulary names how the specification is expressed — the terms used to structure, connect, and identify spec items. Every Catenator spec is written in both languages simultaneously.

---

## Spec structure terms

The language of writing and connecting specs.

| Term | Defines |
|---|---|
| `spec` | the document — the complete structured description of a system or part of one |
| `item` | any named thing within a spec — a system, a model, an actor, an operation |
| `subject` | the system being specified — the thing all descriptors in the spec are describing |
| `object` | an entity the system operates on — what a `model` descriptor item refers to in the world |
| `descriptor` | one of the defined lenses used to specify a system |
| `attribute` | a property of a spec item |
| `value` | what an attribute holds |
| `type` | the classification of an item within its descriptor category |
| `reference` | a pointer from one spec item to another — within or across specs |
| `constraint` | a bounded condition applied to a spec item — internal or external |
| `contract` | the terms of interaction at the boundary between two spec items |
| `relationship` | how two spec items connect — declared, not implied |

---

## Provenance terms

The language of identifying and tracking specs.

| Term | Defines |
|---|---|
| `author` | the person who wrote the spec |
| `organization` | the entity under whose authority the spec is written |
| `publisher` | the entity that makes the spec available |
| `publication` | the act or artifact of making the spec available |
| `version` | which iteration of the spec this is — versions are immutable |
| `date` | when this version was written or published |

Every Catenator spec MUST include `author`, `version`, and `date`. A spec without these is not a valid Catenator spec.

---

**The meta-vocabulary is the stud. It is what makes composability possible across specs, domains, tools, and agents.**
