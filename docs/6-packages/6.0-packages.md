# Packages

A package is a named collection of vocabulary terms for a domain concern. Packages are the unit of vocabulary distribution â€” a spec imports a package to gain access to its terms.

Packages are maintained in `catenator-vocab`. The list below is the canonical package registry.

---

| Package                    | Description                                                                                    |
| -------------------------- | ---------------------------------------------------------------------------------------------- |
| `Professional services`    | Credentials, designations, licensing, scope of practice, and continuing education requirements |
| `Compliance`               | Regulatory requirements, permits, inspections, reporting obligations, and violation handling   |
| `Financial services`       | Revenue recognition, payment terms, financial instruments, credit, and account structures      |
| `Pricing`                  | Price structures, discounting, bundling, rate cards, surcharges, and market pricing mechanisms |
| `Metrics`                  | Measurement frameworks, KPIs, reporting periods, benchmarks, and performance tracking          |
| `Labor`                    | Employment types, compensation structures, scheduling, union terms, and workforce rules        |
| `Platform`                 | Multi-sided market mechanics, marketplace structures, API contracts, and developer relations   |
| `Subscription`             | Recurring revenue models, billing cycles, plan tiers, upgrades, and churn                      |
| `Rights`                   | Intellectual property, licensing terms, permissions, royalties, and territorial rights         |
| `Workflow`                 | Process stages, approval chains, handoffs, status transitions, and queues                      |
| `Inventory`                | Stock units, availability, replenishment, warehousing, and shrinkage                           |
| `Distribution`             | Channels, fulfillment, logistics, last-mile delivery, and supply chain relationships           |
| `Editorial`                | Content lifecycle, review processes, style governance, and publication decisions               |
| `Identity`                 | Authentication, authorization, roles, permissions, and access control                          |
| `Experiential`             | Venue, atmosphere, service encounter, sensory environment, and hospitality                     |
| `Governance`               | Board structures, voting rights, fiduciary obligations, and policy authority                   |
| `Membership`               | Enrollment, tiers, benefits, renewal, and community standing                                   |
| `Events`                   | Scheduling, attendance, ticketing, production, and post-event obligations                      |
| `Regulatory environment`   | Jurisdictional rules, industry-specific law, reporting bodies, and enforcement                 |
| `Commercial relationships` | Contracts, vendor terms, partnership structures, and negotiation mechanics                     |
| `Production`               | Manufacturing, fabrication, quality control, specifications, and tolerances                    |
| `Safety`                   | Hazard identification, protective requirements, incident reporting, and training               |
| `Data`                     | Data models, retention policies, privacy obligations, lineage, and consent                     |
| `Communication`            | Channel management, messaging protocols, notification rules, and audience targeting            |
| `Real estate`              | Occupancy, leasehold terms, zoning, buildout obligations, and property relationships           |
| `Seasonal`                 | Temporal cycles, seasonal pricing, crop years, harvest windows, and off-peak rules             |
